Introduction

Course Objectives

This course has three main learning objectives:

  1. Understand how to define and explain protocols, concepts, and technologies that enable web applications
  2. Develop the ability to analyze, design, and implement web applications that accommodate specific requirements and constraints with regard to issues of usability, performance and security.
  3. Learn how to work as part of a team or individually to explore relevant alternatives and make design and development recommendations.

Technologies Used

We will be using the following technologies in this course. This includes languages, frameworks, libraries, frameworks and APIs. Keep in mind that this is not an exhaustive list:

  1. HTML5
  2. CSS3
  3. JavaScript
  4. jquery
  5. RESTful API
  6. JSON
  7. Node.js
  8. React.js
  9. XML
  10. AJAX

Course Policy

Submission policies: Submission is always due at 11:59pm on MarkUs on day listed on the assignment. Try to submit with ample time left before the deadline to avoid last minute hastles. Late submissions are acceptable to upto two days after the dealine with a 10 % deadline per day. No, we would not accept after the two days unless we get a documented note giving us a good reason and we do not care if your dog ate the note.

Re-marking Policy: Remarking requests are acceptable upto 5 days after a mark is released. No exceptions. You may email the course intructor and schedule an appointment if you are still disatisfied after the remark

Cheating/Plagiarism: Yes, this is big enough to have a seperate paragraph. Anyone caught will be referred to the university's disciplinary committee. You are not only impeding your own learning by doing this but you are also being unfair to the people who put in the time to learn. While asking colleagues for help is not only acceptable but encouraged, copying code or letting someone else copy yours is not. No one needs a black mark on their transcript indicating that he/she got caught cheating.

Resources

Your best resource is the internet itself. This course is more about self learning rather than just using the tools given to you during lectures. A web developer needs to be adaptable and the best way to acquire this ability is doing it from the start. In any case the following websites might prove useful references:

  • W3C: A good tool to validate your work to meet best standards
  • w3schools: An amazing reference to look up CSS and HTML documentation
  • CSS tricks
  • Mozilla developer network
  • jQuery API
  • Node.js API
  • Git Documentation
